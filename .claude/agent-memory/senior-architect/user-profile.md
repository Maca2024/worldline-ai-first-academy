Marco Rink is the technical lead at AetherLink B.V. (operating as RinkGroup). He architects AI-first systems for enterprise clients.

**Preferences:**
- Protocols over frameworks (MCP, A2A — no LangChain)
- Anthropic SDK direct (no framework overhead), prompt caching
- Python + Pydantic for agent logic, Next.js 14 App Router for frontend
- Supabase (EU region) as default DB + auth
- TypeScript strict: true everywhere
- shadcn/ui + Tailwind for UI
- Vercel for Next.js deployments
- Communication in Dutch unless he switches to English

**NS (Nervous System) architecture principles:**
1. Protocols over frameworks
2. Explicit over implicit — traceable state transitions
3. Memory as infrastructure (4 types: episodic, semantic, procedural, working)
4. Cost as first-class concern — model routing, prompt caching
5. Composable at protocol level

**How to apply:** Deliver architecture recommendations that align with these principles. When suggesting tech choices, favor what's already in the AetherLink stack. Do not add complexity without justification.
